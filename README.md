## Project overview

People with disabilities in the United States navigate a deeply fragmented and highly variable policy landscape. Whether a disabled person can access community-based care, employment supports, or housing assistance depends heavily on which state they live in — yet until now, no systematic measure captured the full multidimensional nature of these state policy environments.

This project constructs the **State Disability Policy Index (SDPI)**: a composite index of state disability policy generosity and comprehensiveness built from **36 indicators across five policy domains** covering all 50 states and the District of Columbia (2023–2024 data).

**Headline findings:**
- Top-performing states cluster in the Northeast and Pacific; lower-scoring states cluster in the South
- Rankings are robust across three normalization strategies (*r* ≥ 0.87 across methods)
- State disability policy is **deeply multidimensional**: PC1 explains only 19.1% of variance; 8 components are required to reach 70% — states make genuinely independent structural choices across policy domains

## Five policy domains

| Domain | n | Key indicators |
|--------|---|----------------|
| Economic Security | 11 | SSI payment level, Medicaid eligibility threshold, SSA approval rates, Medicaid Buy-In |
| Community Integration & Housing | 10 | HCBS expenditure ratio, home health aides per 100, Section 811 PRA, housing livability |
| Workforce Inclusion & Education | 6 | VR career/training spending, 14(c) subminimum wage status, UI caregiver provisions, special ed policy |
| Healthcare Access | 6 | ADL Medicaid coverage, medically needy pathway, ARP HCBS investments |
| Civil Rights | 3 | FEMA hazard mitigation, community attendant services, UI caregiver good-cause |

## How to reproduce

### Python (notebooks 01–04)

```bash
# Clone
git clone https://github.com/<your-username>/sdpi-disability-policy-index.git
cd sdpi-disability-policy-index

# Install dependencies
pip install pandas numpy matplotlib seaborn scikit-learn geopandas jupyter nbformat

# Run notebooks in order
jupyter notebook code/
# Execute: 01 → 02 → 03 → 04
```

Notebook 02 writes `data/sdpi_processed.csv`, which notebooks 03 and 04 load.

### R (PCA / FAMD analysis)

```r
install.packages(c("psych", "FactoMineR", "factoextra", "ggplot2", "readr"))
source("code/pca_analysis.R")
```

The R script runs from the repo root directory and writes outputs to `output/`.

> **Note:** The Python notebooks (04) use a Pearson-based PCA approximation for visualization. The R script (`pca_analysis.R`) runs the paper's actual analysis using a polychoric–Pearson mixed correlation matrix via `psych::mixedCor()`.

## Data sources

Data were collected through: (1) systematic scraping of federal and state government websites (CMS, SSA, RSA, HUD, FEMA, state Medicaid agencies); (2) direct verification with regional Medicaid offices; and (3) consultation with disability policy researchers.

Key sources include: AARP 2023 LTSS State Scorecard, KFF Medicaid data, Rehabilitation Services Administration annual reports, HUD Section 811 program data, FEMA hazard mitigation plan status, and SSA administrative data on disability determination rates.
